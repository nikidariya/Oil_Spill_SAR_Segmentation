## О проекте

SAR-снимки (Synthetic Aperture Radar) со спутника Sentinel-1 позволяют обнаруживать нефтяные пятна на поверхности воды независимо от облачности и времени суток — радар использует радиоволны, проходящие сквозь атмосферу, в отличие от оптических снимков. Нефть на радарном снимке выглядит темнее окружающей воды, потому что гасит капиллярные волны и снижает обратное рассеивание сигнала.

Задача усложняется тем, что похожие темные пятна на снимке могут быть вызваны и другими природными явлениями (штиль, биогенные пленки, апвеллинги) — это **look-alike** объекты, которые модель должна научиться отличать от настоящих разливов.

**Цели проекта:**
1. Сравнить три архитектуры сегментации при схожих условиях обучения
2. Выбрать лучшую модель по согласованной командой метрике
3. Реализовать расчет площади разлива по предсказанной маске
4. Собрать прикладной интерфейс (Gradio) для загрузки снимка и получения результата

## Датасет

**Источники:** [Zenodo, record 8346860](https://zenodo.org/records/8346860), [Zenodo, record 8346860](https://zenodo.org/records/8253899), [Zenodo, record 8346860](https://zenodo.org/records/13761290)


Датасет разделен на три части:

| Часть | Содержимое | Объем |
|---|---|---|
| Part 1 | Снимки с нефтяными разливами + маски | 1200 снимков, 1200 масок, ~56 ГБ |
| Part 2 | Снимки без нефти + look-alike объекты + маски | 685*4=2740 объектов, ~64 Гб |
| Part 3 (test) | Тестовая выборка: Oil / No oil / Lookalike + маски | 150*6=900 объектов, 17 Гб |

- **Тип снимков:** SAR (радарные), без необходимости обработки облачности
- **Ground truth:** сегментированные маски (пиксельная разметка нефти)
- **Каналы:** VV и VH поляризация
- **Размер снимка:** 2048×2048 пикселей
- **Разрешение:** 1 пиксель ≈ 10×10 метров (режим Sentinel-1 IW)
- **Дисбаланс классов:** нефть составляет ~1.4% пикселей при соотношении нефть:фон = 1:70

Из-за большого объема данных загрузка происходила через Kaggle API, без локального скачивания.

## Препроцессинг и общие параметры

Чтобы сравнение архитектур было честным, все три модели используют **идентичный препроцессинг**, зафиксированный в `configs/shared_config.json`:

| Параметр | Значение | Комментарий |
|---|---|---|
| Размер патча | 512×512 | Снимок 2048×2048 нарезается на 16 патчей |
| Нормализация | Z-score (VV, VH отдельно) | Статистика рассчитана только по train |
| Оверсэмплинг | 1:1 | Нефтяные патчи vs. фоновые |
| Аугментации | `aug_v1` (без аугментаций) | Подобрано через Optuna на UNet — не дало прироста |
| Loss | BCE + Dice | `bce_weight` и `pos_weight` подобраны через Optuna |
| Энкодер | ResNet-34 | Единый для UNet и DeepLabV3+ |
| Train/Val/Test split | Зафиксирован в CSV | Одинаковые патчи у всех участников, train/val — 80/20 |

**Известное ограничение:** общий конфиг был оптимизирован на UNet. Это не идеально архитектурно-нейтральная настройка — SegFormer теоретически мог выиграть от других аугментаций. Эта оговорка сделана сознательно и документируется как ограничение исследования.

Каждая модель использует собственный поиск гиперпараметров обучения (optimizer, scheduler, learning rate, weight decay) через Optuna — здесь архитектуры не ограничены общим конфигом.

## Архитектуры и результаты

### Алгоритм обучения (общий для всех моделей)

1. **Этап 1** — быстрая проверка пайплайна на базовой конфигурации
2. **Этап 2, Шаг 1** — Optuna: подбор optimizer / scheduler / lr / weight_decay
3. **Этап 2, Шаг 2** — топ-3 конфига обучаются дольше, выбор лучшего по среднему `val_oil_iou` последних 3 эпох
5. **Этап 2, Шаг 3** — финальное обучение на полном количестве эпох
6. **Этап 2, Шаг 4** — финальная оценка: patch-level и scene-level метрики на val и test

### UNet (базовая модель)

*Автор: Дарья*

Классический энкодер-декодер с skip-connections на каждом уровне. Локально анализирует каждый патч, восстанавливая мелкие детали границ объектов через прямую передачу feature map с энкодера на декодер.

| Метрика | Val | Test |
|---|---|---|
| Oil IoU | 0.6078 | 0.6440 |
| Oil Dice | 0.6574 | 0.6785 |
| Background IoU | 0.9763 | 0.9379 |
| Mean IoU | 0.7920 | 0.7909 |
| Scene Oil IoU | 0.5629 | 0.5910 |
| Scene Oil Dice | 0.6194 | 0.6333 |

### DeepLabV3+ (улучшенный бейзлайн)

*Автор: Жанслу*

Использует модуль ASPP (Atrous Spatial Pyramid Pooling) — параллельные свёртки с разными dilation rate (6, 12, 18), позволяющие модели видеть контекст одновременно на нескольких масштабах. Это особенно важно для нефтяных пятен, которые сильно различаются по размеру — от узкой полосы до крупного разлива.

| Метрика | Val | Test |
|---|---|---|
| Oil IoU | 0.6358 | 0.6532 |
| Oil Dice | 0.6841 | 0.6828 |
| Background IoU | 0.9882 | 0.9749 |
| Mean IoU | 0.8120 | 0.8140 |
| Scene Oil IoU | 0.5933 | 0.5742 |
| Scene Oil Dice | 0.6486 | 0.6114 |

### SegFormer (трансформер)

*Автор: Ольга*

Архитектура на основе Vision Transformer — рассматривает снимок целиком через self-attention, а не локально по патчам. В теории это позволяет улавливать дальние зависимости между удаленными частями изображения, что полезно для крупных разливов, пересекающих границы нескольких патчей.

| Метрика | Val | Test |
|---|---|---|
| Oil IoU | 0.5503 | 0.5480 |
| Oil Dice | 0.6027 | 0.5856 |
| Background IoU | 0.9915 | 0.9754 |
| Mean IoU | 0.7709 | 0.7617 |
| Scene Oil IoU | 0.5503 | 0.5480 |
| Scene Oil Dice | 0.6027 | 0.5856 |

### Итоговое сравнение

| Архитектура | Test Oil IoU | Test Oil Dice | Test Scene Oil IoU |
|---|---|---|---|
| UNet | 0.6440 | 0.6785 | 0.5910 |

## Метрики

### Мониторинг во время обучения (по эпохам)

- `train_loss` — функция потерь на train, усредненная по батчам за эпоху
- `val_loss` — функция потерь на валидации
- `val_oil_iou` — IoU по классу нефти на валидации (главная метрика проекта)
- `val_background_iou` — IoU по классу фона (если модель ложно предсказывает «всё фон», эта метрика останется высокой при падении `val_oil_iou`)

### Итоговые метрики (после финального обучения, на лучшем чекпоинте)

**Patch-level** (на отдельных патчах 512×512):
- `val_oil_iou`, `val_background_iou`, `val_mean_iou`
- `val_oil_dice` — Dice по классу нефти
- Аналогичные метрики для test

**Scene-level** (на полных снимках 2048×2048, патчи собираются обратно перед расчетом):
- `val_scene_oil_iou`, `val_scene_oil_dice`
- `test_scene_oil_iou`, `test_scene_oil_dice`

Scene-level метрика рассчитывается отдельно, потому что в прикладном сценарии (загрузка снимка целиком в Gradio) важна точность по снимку, а не по случайному патчу — каждый снимок при усреднении учитывается как один голос, независимо от размера пятна на нем.

### Почему IoU, а не Accuracy

При дисбалансе классов accuracy неинформативна — модель, предсказывающая «всё фон», получит высокий accuracy при нулевой практической пользе. IoU по классу нефти отдельно от фона устраняет эту проблему.

### Выбор лучшей модели

Лучшая эпоха в рамках одного запуска обучения выбирается по пиковому значению `val_oil_iou` одной эпохи.

## Приложение Gradio

Прикладной интерфейс позволяет:
1. Загрузить SAR-снимок
2. Получить предсказание: есть ли нефтяной разлив
3. Получить оценку площади разлива в км² / га (по количеству нефтяных пикселей × площадь пикселя, с учетом масштаба снимка)

*Автор: Ольга*

Запуск:
```bash
cd app
pip install -r requirements.txt
python gradio_app.py
```

## Ограничения

- Общий конфиг препроцессинга (loss, аугментации, oversampling) подбирался на UNet и применен ко всем архитектурам без повторной оптимизации под каждую
- Подбор гиперпараметров обучения (Optuna) проводился с разным количеством trials/эпох между моделями из-за различий в скорости обучения архитектур и доступных ресурсах
- Test-выборка использовалась только один раз для каждой финальной модели — метрики не корректировались по результатам теста
